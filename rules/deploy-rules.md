# 部署规范

## 部署流程

1. 本地验证：功能分支开发完成，本地测试通过，无敏感文件
2. 推送远程：合并到 main（或对应部署分支），push 到 GitHub
3. 服务器拉取：ssh 登录，进入项目目录，`git pull`
4. 重启服务：按 deploy.md 记录的命令执行
5. 部署验证：确认服务正常运行

## 环境

- ECS（阿里云 Linux 3）：`ssh ecs`（配置在 ~/.ssh/config）

## deploy.md

项目若有值得沉淀的部署流程，在根目录维护 deploy.md。标准是清晰可执行——任何人读完能照着跑起来。

常见内容按需取舍：部署方式、服务器地址与路径、启动/重启命令、依赖的环境变量、常见故障处理。

不提交到 git——含服务器地址等敏感信息，创建时加入 .gitignore。

部署前读 deploy.md 识别方式，无则按项目实际直接部署。首次部署后新建 deploy.md，后续流程或命令变化时同步更新。
